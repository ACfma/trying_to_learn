Quando si fa il commit si deve usare il -m "messaggio da inserire" perchè altrimenti si va dentro vim (?)

Una volta fatto il commit git mi da un hash prima del nome del file.

Poi va pushato il file. IL PUSH AGISCE SU TUTTO CIò CHE é STATO COMMITTATO.

Il pull invece serve per vedere i cambiamentio ed è opportuno farlo prima di iniziare a lavorare.

git status mi dice lo status del repositorio, se c'è qualcosa di verde è nuovo. Se rosso è untrucked e devo fare git add per tracciarlo (da capire cosa vuol dire).
Una volta verde il file è pronto per il commit.

Se modifico un file già committato posso utilizzare add per aggiungere le modifiche o restore per tornare alla versione precedentemente committata.

ATTENZIONE: SEMPRE METTERE NOMI NON FRAINTENDIBILI.

Anche i branch vanno committati!

git branch *nome* crea un nuovo branch.
git checkout *nome* mi fa passare sul branch creato.
Per creareil branch online su github devi usare git push origin *nome branch*
Per pushare il branch sul master (detto anche merge) fare git push origin *nome branch*:*nome master*
git branch -d *nome branch* per eliminare il branch. IN GENERALE -d  ELIMINA L'ELIMINABILE.


La pull request è l'inizio del processo per pushare un branch sul master.(cioè inizio delle modifiche che andranno a far parte del master)
